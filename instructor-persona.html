Instructor Persona

Persona

Name: Monty Fibonacci

Age: 71

Profession

Coder extraordinaire; javascript ninja; deep dive master instructor.

Technology

Monty is a linux superuser. He teaches both in person and remotely through a variety of operating systems (linux, windows, macOS) as well all modern browsers for different functions.

Attitudes and Behaviors

He codes before, during, and after work, often creating fully-functioning mobile games from scratch to deployment in 3 hours and 50 minutes. However, he does not know how to code a voting app.

Frustrations and Needs

With every new cohort comes the need for capstone project ideas to be listed, filtered, selected, and assigned. The instructor currently has to use 4 different applications and multiple screens causing delays in screen sharing, data loss, and an inefficient process for topic selection.

Goals

Monty wants an application that allows the cohort members/instructors to list all topic ideas, filter them out, allow students to vote, and store valid ideas in a database for future use.

User Story

As an Instructor, I need an application that allows an easy interface to list, delete, and vote on capstone ideas.
